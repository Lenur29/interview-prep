# Модуль Images (Зображення)

## Призначення

Модуль `Images` призначений для комплексного управління графічними файлами в системі `ugi-cloud`. Його головна задача — зберігати базові метадані оригінальних зображень та зручно керувати їхніми варіаціями/форматами (Renditions). Це дозволяє системі оптимізувати трафік та віддавати клієнтам зображення найбільш відповідного розміру та якості.

## Роль у проєкті

Модуль виконує роль централізованого медіа-сховища. Він спроєктований як гнучкий будівельний блок: завдяки експортуванню ключових сервісів (`ImagesService` та `ImageRenditionsService`) у файлі `images.module.ts`, він надає свій публічний інтерфейс іншим частинам додатку. Інші модулі можуть легко імпортувати його для безпечної прив'язки графічних файлів до своїх сутностей (наприклад, публікацій чи профілів користувачів).

## Як він працює (Структура модуля)

Модуль побудований за принципами чистої архітектури NestJS та використовує **GraphQL** замість класичного REST API для обміну даними. Основні складові:

### `images.module.ts` — Головний конфігураційний файл

Об'єднує всі компоненти через механізм Dependency Injection та реєструє сутності бази даних через `TypeOrmModule.forFeature([Image, ImageRendition])`. Ключово, що сервіси (`ImagesService`, `ImageRenditionsService`) додано до масиву `exports` — це робить їх доступними для інших модулів проєкту.

### `image.entity.ts` — Сутність бази даних

Описує структуру таблиці `images` у базі даних (TypeORM). Ключові особливості:

- Використовує механізм **спадкування таблиць** (`@TableInheritance`) — в одній таблиці можуть зберігатись різні підтипи зображень.
- Містить зв'язок **«Один-до-багатьох»** (`@OneToMany`) із варіаціями зображень (`ImageRendition`). Параметр `eager: true` означає, що варіації завантажуються автоматично разом із оригіналом.
- Має вбудовану «розумну» логіку — метод `get url()`, який автоматично підбирає найкращу доступну якість зображення за пріоритетом: `MAIN_2X → MAIN → MAIN_LEGACY → ORIGINAL`.

### `images.service.ts` — Сервіс (бізнес-логіка)

Містить головну бізнес-логіку роботи з зображеннями. Усі методи супроводжуються детальним логуванням та обробкою контексту (`ServiceMethodContext`). Ключові методи:

| Метод                   | Опис                                                      |
| ----------------------- | --------------------------------------------------------- |
| `getOne(id)`            | Знаходить зображення за ID або повертає `null`            |
| `getOneOrFail(id, ctx)` | Знаходить зображення або кидає `NotFoundError`            |
| `getUrlById(id, type?)` | Повертає URL зображення за ID та типом варіації           |
| `create(opts, ctx)`     | Створює зображення та каскадно всі його варіації          |
| `update(opts, ctx)`     | Оновлює метадані зображення                               |
| `delete(id, ctx)`       | Видаляє зображення та **каскадно** всі пов'язані варіації |

### `images.resolver.ts` — GraphQL Резолвер

GraphQL-інтерфейс модуля (аналог контролера у REST). Жорстко захищений двома гардами:

- **`GraphQLAuthGuard`** — дозволяє доступ лише авторизованим користувачам.
- **`GraphQLPermissionsGuard`** — кожна дія вимагає конкретного системного дозволу (наприклад, `tl:core:images:delete`).

### Підмодуль `image-renditions/`

Відповідає за управління варіаціями зображень (мініатюри, різні формати, розміри). Містить власні сутність (`image-rendition.entity.ts`), сервіс та резолвер за аналогічною структурою.

### Папка `types/`

Містить DTO (Data Transfer Objects) та інтерфейси для строгої типізації вхідних і вихідних даних GraphQL (наприклад, `CreateImageInput`, `UpdateImageInput`, `DeleteImagePayload`).

## Основні ендпоінти (GraphQL API)

Оскільки модуль використовує GraphQL, він надає наступні точки доступу:

### Queries (Отримання даних)

| Query                | Опис                                           | Необхідний дозвіл    |
| -------------------- | ---------------------------------------------- | -------------------- |
| `image(id: String!)` | Отримати зображення та всі його варіації за ID | `tl:core:images:get` |

### Mutations (Зміна даних)

| Mutation                                | Опис                                         | Необхідний дозвіл       |
| --------------------------------------- | -------------------------------------------- | ----------------------- |
| `createImage(input: CreateImageInput!)` | Створити нове зображення разом із варіаціями | `tl:core:images:create` |
| `updateImage(input: UpdateImageInput!)` | Оновити метадані існуючого зображення        | `tl:core:images:update` |
| `deleteImage(input: DeleteImageInput!)` | Видалити зображення та всі його варіації     | `tl:core:images:delete` |
