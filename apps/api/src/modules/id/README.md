# Модуль ID (IdModule)

## Призначення

Модуль **ID** забезпечує централізовану систему генерування унікальних ідентифікаторів для всіх сутностей в додатку. Основна задача модуля — генерувати читаємі та безпечні ID, які могли б бути легко розпізнані під час розробки та у журналах, але й безпечні для використання в продакшені.

## Роль у проєкті

IdModule є глобальним сервісом (з декоратором `@Global()`), що означає, що він автоматично доступний в усіх модулях додатку без явного імпорту. Модуль вирішує важливу задачу:

- **Забезпечує консистентність**: Усі модулі (users, roles, students, binary-files, images тощо) використовують єдиний központний спосіб генерування ID
- **Контекстна інформація в ID**: Ідентифікатори містять префікс сутності та скорочену назву додатку, що допомагає при отладці та аналізі логів  
- **Середовищна диференціація**: В залежності від середовища (development, test, stage, production) ID містять відповідний префікс для явного розрізнення
- **Ізоляція від інших модулів**: Це утилітарний модуль без власних контролерів та API-рутів

## Як він працює

### Основна логіка

#### IdService
Серцевина модуля — це сервіс `IdService`, який надає два основних методи:

1. **generateEntityId(prefix: string, app?: string): string**
   - Генерує унікальний ID для сутностей в базі даних
   - Параметри:
     - `prefix` — 1-2 символів, префікс типу сутності (наприклад: 'u' для користувачів, 'r' для ролей)
     - `app` — (необов'язково) скорочена назва додатку; якщо не вказано, використовується значення із конфігу
   - Повертає ID у форматі: `{app}{prefix}{delimiter}{random}` (наприклад: `usu:dev:xxxxxxxxxxx`)
   - Розділювач змінюється залежно від середовища:
     - Development: `:dev:`
     - Test: `:test:`
     - Staging: `:stage:`
     - Production: `:`

2. **generateId(size?: number): string**
   - Генерує простий випадковий рядок без структури
   - Параметр `size` — довжина рядка (за замовчуванням 11 символів)
   - Результат: просто випадкова послідовність символів (наприклад: `xxxxxxxxxxx`)

### Ключові файли

- **id.module.ts** — NestJS модуль, що експортує IdService як глобальний провайдер
- **id.service.ts** — Сервіс із методами генерування ID та логікою обробки середовища

### Залежності

- `AppConfig` — конфігурація додатку для отримання поточного середовища та скорочене назви
- `AppEnv` — enum із можливими середовищами (DEVELOPMENT, TEST, STAGE та інші)
- `@pcg/text-kit` — утиліти для генерування випадкових рядків
- `@nestjs/config` — система конфігурації NestJS

### Інтеграція з іншими модулями

Інші модулі (users, roles, students тощо) інжектують `IdService` та використовують його під час:
- Виконання операцій створення сутностей
- Генерування ID для нових записів в базі даних

```typescript
// Приклад інтеграції в іншому модулі:
import { IdService } from '../id/id.service';

@Injectable()
export class UsersService {
  constructor(private readonly idService: IdService) {}

  createUser(data: CreateUserDto) {
    const userId = this.idService.generateEntityId('u'); // Результат: наприклад 'jsu:dev:abc123xyz'
    // Далі використовується userId для створення користувача
  }
}
```

## Ендпоінти

Модуль **не надає жодних API-рутів або ендпоінтів**. Це утилітарний внутрішній сервіс, який використовується іншими модулями на внутрішньому рівні для генерування ID при створенні та обробці сутностей. Доступ до нього можливий тільки в коді через інжекцію залежностей.
