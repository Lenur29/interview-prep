# Roles Module

## Призначення
Модуль `roles` відповідає за опис системних ролей, їхніх типів та прав доступу. Він надає централізовану інформацію про ролі, які використовуються у авторизації та контролі доступу в API.

## Роль у проєкті
Цей модуль інтегрується в основну GraphQL-архітектуру бекенду. Він надає дані про ролі, які використовуються іншими модулями, зокрема системою авторизації та захисту ресурсів, а також загальними GraphQL-резольверами, які можуть потребувати список ролей для відображення або валідації.

## Як він працює
- `roles.module.ts` реєструє `RolesResolver` і `RolesService` як провайдерів модуля.
- `roles.service.ts` працює з реєстром ролей `ROLES_REGISTRY` з модуля `@/permissions/roles.js` і формує масив об'єктів `Role`.
- `role.ts` описує GraphQL-тип `Role`, який містить поля: `id`, `title`, `type`, `description` і `permissions`.
- `roles.resolver.ts` визначає GraphQL-запит `roles`, який повертає перелік усіх ролей через сервіс.

## Основні ендпоінти
Модуль надає GraphQL-запит:

- `roles`: повертає масив ролей з полями `id`, `title`, `type`, `description` та `permissions`.
